# Aul2MIRAI 生成用エイリアスカタログ

この文書は、ユーザーが用意したAviUtl2サンプルから確認した、オブジェクトおよびフィルターの生成用エイリアスを記録する。外部操作コマンドは[`AI_USAGE.md`](AI_USAGE.md)、通信と内部処理は[`PIPE_INTERFACE.md`](PIPE_INTERFACE.md)、実機確認の時系列は[`HISTORY.md`](HISTORY.md)を参照する。

## 運用規則

- 外部向けの一般削除は扱わない。削除は、フィルター追加時の内部置換と、失敗時に元のオブジェクトへ戻すロールバックにだけ使用する。

- サンプルで確認できた項目名、値形式、エフェクト順だけを記録し、未確認の値を推測しない。
- AviUtl2本体、対象プラグイン、サンプルファイルのバージョンまたは確認日を記録する。
- 初期値と変更値を区別し、非初期値を保持して生成できた場合は明記する。
- エイリアス生成後は、配置、種類、素材パス、エフェクト順、全設定値、有効・ロック状態、中間点、トラック情報を元または期待値と比較する。
- 生成時に値が正規化された場合は、成功例として登録せず「要補正」または「未対応」とする。
- 複数オブジェクト間のグループ・連結情報は公開SDKから取得できないため、単一オブジェクトエイリアスだけで保持できると判断しない。

確認状態は次の語を使用する。

| 状態 | 意味 |
| --- | --- |
| 生成確認済み | 空き位置への生成と詳細比較に成功 |
| 同位置置換確認済み | 元を削除し、同位置への生成と詳細比較に成功 |
| 設定変更確認済み | SDKによる生成後の設定値変更と再取得に成功 |
| 要補正 | エイリアス生成だけでは値が一致しない |
| 未確認 | サンプルはあるが生成・比較を完了していない |

## 共通構造

単一オブジェクトでは、オブジェクト本体を`[0]`、所有するエフェクトを`[0.0]`、`[0.1]`のような連番ブロックで表す。各エフェクトブロックの先頭には`effect.name`を置き、続けてそのエフェクトの全設定項目をエイリアスと同じ文字列表現で記述する。

既存エフェクトを繰り返す場合は、元の最後の同名ブロックを複製し、現在の最大エフェクト番号に1を加えたブロックとして末尾へ追加する。既存ブロックの順序と内容は変更しない。

例:

```ini
[0.2]
effect.name=色調補正
明るさ=110.00
コントラスト=100.00
色相=0.00
輝度=100.00
彩度=100.00
飽和する=1

[0.3]
effect.name=色調補正
明るさ=110.00
コントラスト=100.00
色相=0.00
輝度=100.00
彩度=100.00
飽和する=1
```

## 標準オブジェクト: 図形

- 確認状態: 全8種類の生成確認済み、円の同位置置換確認済み
- 確認環境: AviUtl2 v2.1.0、1920×1080、30 fps、44100 Hz
- 初期値: `円`（種類リスト先頭の`背景`ではない）
- 確認済みの種類リスト順: `背景`、`円`、`四角形`、`三角形`、`五角形`、`六角形`、`星型`、`ハート`
- 通常のエフェクト順: `図形`、`標準描画`

全種類で`図形の種類`以外の設定値は同一だった。

```ini
[0.0]
effect.name=図形
図形の種類=<確認済みの図形名>
サイズ=100
縦横比=0.00
ライン幅=4000
色=ffffff
角を丸くする=0
```

初期値の円:

```ini
[0.0]
effect.name=図形
図形の種類=円
サイズ=100
縦横比=0.00
ライン幅=4000
色=ffffff
角を丸くする=0
```

| 1 | `背景` | レイヤー0・117～197 | レイヤー1・117～197 | リスト先頭だが初期値ではない |
| 2 | `円` | レイヤー0・198～278 | レイヤー1・198～278 | 新規作成時の初期値 |

## 標準フィルター: スクリプト制御

- 確認状態: Lua円運動を含む図形オブジェクト内部での生成確認済み
- 確認環境: AviUtl2 v2.1.0、1920×1080、30 fps、44100 Hz
- 元サンプル: レイヤー0・9012～9212フレーム
- 生成確認: レイヤー0・9213～9413フレーム
- 確認済みのエフェクト順: `図形`、`標準描画`、`スクリプト制御`

```ini
[0.2]
effect.name=スクリプト制御
スクリプト=-- 円移動\nl = 200\n\n-- 半径\nr = 360 * obj.time / 2\n\nobj.ox = obj.ox + math.sin(r * math.pi / 180) * l\nobj.oy = obj.oy - math.cos(r * math.pi / 180) * l\nobj.rz = obj.rz + r
```

確認結果:

- `スクリプト制御`は通常オブジェクトの出力エフェクトより後ろに置くフィルターブロックとして生成できた。今回の図形では、`図形`、`標準描画`、`スクリプト制御`の順である。
- 設定項目は`スクリプト`1件だけで、トラック情報はない。元サンプルと生成物はいずれも有効、ロックなしだった。
- Lua本文の改行は、エイリアスの1行内でリテラルの`\n`として記述する。JSON要求ではJSON自身のエスケープも必要になるため、送信言語側でバックスラッシュを保持する。
- 完全エイリアスを`create_object_from_alias`で生成し、配置、主効果、素材パス、`図形`、`標準描画`、`スクリプト制御`の順序、全設定項目名と値の生成直後検証に成功した。
- 元と生成物について相対時刻0、0.5、1.0、2.0秒をPipeキャプチャし、4組すべてBMPのSHA-256が一致した。Lua本文は文字列として保持されるだけでなく、同じ時間基準で実行された。
- 現在確認したのは配置済みLuaの読取と、同一Luaを含む完全エイリアスの新規生成である。AIが新規作成した別内容のLua、Pipeによる本文変更、独立した`フィルタオブジェクト`としての配置、エラー時の挙動は未確認。

## 標準オブジェクト: フィルタオブジェクト

- 確認状態: 非初期値の色調補正を持つ単体生成確認済み、直下配置による元サンプルへの作用はユーザー確認済み
- 確認環境: AviUtl2 v2.1.0、1920×1080、30 fps、44100 Hz
- 対象サンプル: `色=ff0000`の円と、その直下に配置した独立した色調補正
- 確認済みのエフェクト順: `フィルタオブジェクト`、`色調補正`

```ini
[0.0]
effect.name=フィルタオブジェクト

[0.1]
effect.name=色調補正
明るさ=51.60
コントラスト=88.30
色相=0.00
輝度=100.00
彩度=100.00
飽和する=1
```

配置と確認結果:

- 元の赤い円はレイヤー1・1095～1495フレーム、フィルタオブジェクトは直下のレイヤー2・1230～1630フレームだった。両者が重なる1230～1495フレームで色調補正が円へ作用することをユーザーが画面上で確認した。
- 赤い円は`図形の種類=円`、`色=ff0000`で、その他は確認済みの図形・標準描画初期値だった。
- フィルタオブジェクト本体には設定項目がなく、有効・ロックありだった。後段の色調補正は有効・ロックなしだった。
- フィルタオブジェクトをレイヤー4・1230～1630フレームへ再生成し、401フレーム、エフェクト順、非初期値を含む全設定値、状態、中間点、トラック情報が一致して`recreation_verification.verified=true`となった。
- 赤い円をレイヤー3・1095～1495フレームへ再生成し、401フレームの全比較項目が一致して`recreation_verification.verified=true`となった。レイヤー差1と元の時間範囲を保持してペアを配置した。
- 外部向け種類`filter_object`として分類する処理を追加した。再ビルド・再起動後、独立配置したローテーションと反転の取得で`object_type=filter_object`を実機確認した。
- 公開SDKからフィルタオブジェクトと作用対象の明示的な関連情報は取得できない。複製ペアの画面上の作用、複数候補が上下にある場合の対象決定、時間範囲外の挙動、複数レイヤーへの作用、他フィルターとの組み合わせは未確認。

### 独立配置とオブジェクト内部追加の互換性

同じ赤い円に対し、独立したフィルタオブジェクトを直下に置く構成と、円自身の末尾へ色調補正を追加する構成を並べて取得した。

| 構成 | エフェクト順 | 色調補正の設定 |
| --- | --- | --- |
| 独立配置 | `フィルタオブジェクト`、`色調補正` | 明るさ`51.60`、コントラスト`88.30`、他は初期値 |
| 円内部 | `図形`、`標準描画`、`色調補正` | 明るさ`100.00`、コントラスト`100.00`、他は初期値 |

- 両方式の色調補正は、項目名、項目順、値形式、トラック形式、有効・ロック状態が同一だった。現在値の差は配置済みサンプルの設定差であり、所有する本体による項目体系の差ではない。
- AviUtl2ではフィルターの初期値と利用項目は両方式で共通であり、独立配置では通常オブジェクトの代わりに`フィルタオブジェクト`が所有本体になるだけであることをユーザー確認として記録した。
- 円内部に色調補正を持つ1個体をレイヤー0・1266～1666フレームからレイヤー2の同区間へ複製し、401フレームの全比較項目が一致して`recreation_verification.verified=true`となった。
- 独立構成は、円をレイヤー0からレイヤー2、フィルタオブジェクトをレイヤー1からレイヤー3の865～1265フレームへ複製した。2個体とも全比較項目が一致し、レイヤー差1と同一時間範囲を保持した。

学習規則は次のように確定する。

- 独立フィルターとして生成する場合は、`[0.0] フィルタオブジェクト`の後ろへ学習したフィルターブロックを置く。
- 通常オブジェクト内部へ追加する場合は、`フィルタオブジェクト`ブロックを除外し、学習したフィルターブロックだけを対象オブジェクトのエフェクト末尾へ追加する。
- フィルターブロックの項目と値は変換せず、そのまま使用する。生成後の詳細比較は両方式とも引き続き必須とする。

この構造により、対象を削除しない独立フィルタオブジェクト方式と、対象内部へ追加する同位置置換方式の両方へ、同じ学習済みフィルターブロックを利用できる。

## 標準フィルター: 色調補正

- 確認状態: 初期値・非初期値の生成確認済み、同位置置換確認済み、設定変更確認済み
- 確認環境: AviUtl2 v2.1.0
- サンプル: `aup2\filter_sample.aup2`、未保存プロジェクト上の独立フィルター初期値
- 対象: 図形オブジェクト
- サンプルの非初期値: `明るさ=110.00`（初期値100.00から変更）

```ini
[0.n]
effect.name=色調補正
明るさ=110.00
コントラスト=100.00
色相=0.00
輝度=100.00
彩度=100.00
飽和する=1
```

| 項目 | サンプル値 | 値形式 | 確認内容 |
| --- | --- | --- | --- |
| 明るさ | `110.00` | 小数文字列 | 非初期値を保持して生成 |
| コントラスト | `100.00` | 小数文字列 | 2段目だけ`110.00`へ変更して再取得 |
| 色相 | `0.00` | 小数文字列 | 生成一致 |
| 輝度 | `100.00` | 小数文字列 | 生成一致 |
| 彩度 | `100.00` | 小数文字列 | 生成一致 |
| 飽和する | `1` | 真偽相当の整数文字列 | 生成一致 |

確認結果:

- 独立した初期値フィルターをレイヤー0・0～400フレームからレイヤー1の同区間へ再生成し、初期値、状態、中間点、トラック情報が一致して`recreation_verification.verified=true`となった。
- 空きレイヤーへの再生成で`明るさ=110.00`を保持した。
- 同じブロックを末尾へ追加し、色調補正2段を生成できた。
- 元を削除して同じレイヤー・フレームへ生成し、オブジェクト数、配置、選択、フォーカスを維持した。
- 2段とも明るさ`110.00`、コントラスト`100.00`で生成した。
- 同名エフェクトを出現順で指定し、2段目のコントラストだけを`100.00`から`110.00`へ変更できた。
