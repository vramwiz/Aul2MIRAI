# AviUtl2 Lua・スクリプト制御 学習ノート

この文書は、AviUtl2の`スクリプト制御`とLuaスクリプトをAIが安全に利用するための正本である。公式説明から読み取った仕様と、Aul2MIRAIのNamed Pipe経由で実機確認した結果を区別して記録する。

生成に使う完全エイリアスは[`ALIAS_CATALOG.md`](ALIAS_CATALOG.md)、Pipeの操作方法は[`AI_USAGE.md`](AI_USAGE.md)、映像効果の実測は[`EFFECT_LEARNING.md`](EFFECT_LEARNING.md)を参照する。

## 参照元

- 公式説明: `D:\aviutl2_v2.1.0\lua.txt`
- 対象: AviUtl2 v2.1.0
- 読取日: 2026-07-24
- ファイル形式: UTF-8
- SHA-256: `2007343F256D9B5803F91FE7AD1245369B9E18865F823FAFDBEA87D8178AE6E1`
- Lua: 5.1.4に5.1.4-2パッチを適用した付属`lua.dll`
- LuaJIT: 2.1の付属`luaJIT.dll`

本書は実作業で必要な要点を整理したものであり、関数の全引数やピクセル・シェーダー処理の詳細は公式`lua.txt`を正本とする。AviUtl2の更新後はファイルのハッシュと更新履歴を確認し、本書の差分を見直す。

## まず守る規則

- `スクリプト制御`で利用できる標準ライブラリは`table`、`string`、`math`である。`os`、`debug`、`ffi.C`は利用できない。
- AIが生成するインラインLuaでは、確認済みになるまで外部ファイル、フォルダー、モジュール、シェーダー、音声、ピクセルバッファ、`global`共有状態を使用しない。
- 無限ループ、極端に大きいループ、再帰、巨大な文字列やテーブル、フレームごとの高負荷処理を生成しない。
- 座標、角度、拡大率、不透明度などは、既存値へ加算または乗算して適用する。理由なく固定値で上書きしない。
- 時間変化は原則として`obj.time`を使い、周期、振幅、単位を式から読み取れる形にする。
- スクリプト本文の変更は、Named Pipeの`preview_set_object_parameters`で検証してから`set_object_parameters`で適用する。
- 構文エラーや実行時エラーの試験は複製した一時オブジェクトで行い、復旧確認後にエラー状態を残さない。
- AviUtl2の操作はNamed Pipeだけで行う。画面座標によるマウス・キーボード自動操作は使用しない。

## スクリプトの利用場所

Luaは次の場所で利用できる。

- 標準フィルターの`スクリプト制御`
- スクリプトファイル: `*.anm2`、`*.obj2`、`*.cam2`、`*.scn2`、`*.tra2`
- テキストオブジェクトのスクリプト制御文字

新形式のスクリプトファイルはUTF-8で、`ProgramData\aviutl2\Script`またはその1階層下へ配置する。旧形式の`*.anm`、`*.obj`、`*.cam`、`*.scn`、`*.tra`も利用できるが、文字コードはShift_JISであり、一部仕様が異なる。

「キャッシュを破棄」でスクリプトとシェーダーは再読み込みされるが、設定項目の変更は反映されない。スクリプトから別のスクリプトを呼ぶ構成は正常に動作しない場合がある。

## `スクリプト制御`の配置と保存形式

2026-07-24時点の実機確認では、通常オブジェクトに対する`スクリプト制御`は出力エフェクトの後ろへ置く。

```text
図形
標準描画
スクリプト制御
```

設定項目は`スクリプト`1件である。完全エイリアスではLua本文を1行の値として保存し、改行をリテラルの`\n`で表す。取得した値を表示・編集するときは、JSONのエスケープとエイリアス内の`\n`を混同しない。

## 基本変数

### 変更可能なオブジェクト状態

| 分類 | 変数 | 意味 |
| --- | --- | --- |
| 相対座標 | `obj.ox`、`obj.oy`、`obj.oz` | 基準座標からの相対X、Y、Z |
| 回転 | `obj.rx`、`obj.ry`、`obj.rz` | X、Y、Z軸回転角度。360.0で1回転 |
| 中心 | `obj.cx`、`obj.cy`、`obj.cz` | 中心の相対座標 |
| 拡大 | `obj.sx`、`obj.sy`、`obj.sz` | 軸ごとの拡大率。1.0で等倍 |
| 互換拡大 | `obj.zoom` | 拡大率。1.0で等倍 |
| 互換縦横比 | `obj.aspect` | `-1.0`～`1.0`。正は横縮小、負は縦縮小 |
| 不透明度 | `obj.alpha` | `0.0`で透明、`1.0`で不透明 |

これらは、その時点のオブジェクト設定値であり、`標準描画`などの出力項目の保存値そのものではない。スクリプトで変化させても、出力項目の設定値を書き換えたことにはならない。

### 読取専用の状態

| 分類 | 変数 | 意味 |
| --- | --- | --- |
| 表示基準座標 | `obj.x`、`obj.y`、`obj.z` | 表示の基準座標 |
| 画像・画面 | `obj.w`、`obj.h`、`obj.screen_w`、`obj.screen_h` | オブジェクトと画面のピクセル寸法 |
| 時間 | `obj.framerate`、`obj.frame`、`obj.time` | フレームレート、オブジェクト基準フレーム、秒 |
| 長さ | `obj.totalframe`、`obj.totaltime` | オブジェクトの総フレーム数と総秒数 |
| 配置 | `obj.layer` | 描画対象オブジェクトのレイヤー |
| 複数描画 | `obj.index`、`obj.num` | 個別オブジェクトの番号と個数 |
| 識別 | `obj.id`、`obj.effect_id` | 起動中だけ有効なオブジェクトと対象エフェクトの固有ID |

`obj.id`と`obj.effect_id`はAviUtl2の起動ごとに変わり得るため、永続識別子として保存しない。シーンチェンジ対象では`obj.frame`と`obj.time`が総時間を超える場合がある。

### 共有状態

`global.xxx`はスクリプト間で共有されるテーブルで、値はバイナリセーフな文字列として保持される。別オブジェクトとの隠れた依存や再現性低下につながるため、AI生成では明示的に必要な場合だけ使う。

## 最小パターン

### 一定速度の移動

```lua
obj.ox = obj.ox + obj.time * 100
```

開始位置を基準に毎秒100px右へ移動する。固定フレームレートを仮定して`obj.frame`を直接掛けるより、秒単位の`obj.time`を使う。

### 回転

```lua
obj.rz = obj.rz + obj.time * 90
```

毎秒90度でZ軸回転する。

### 周期運動

```lua
local radius = 200
local period = 2
local angle = obj.time * math.pi * 2 / period

obj.ox = obj.ox + math.sin(angle) * radius
obj.oy = obj.oy - math.cos(angle) * radius
```

半径200px、2秒周期で、上から時計回りに円運動する。角度は内部でラジアンへ統一すると式が読みやすい。

### 不透明度

```lua
obj.alpha = obj.alpha * 0.5
```

現在の不透明度を半分にする。`obj.alpha = 0.5`と固定すると、それ以前の処理結果を捨てる点に注意する。

## スクリプトファイルの設定項目

スクリプトファイルの先頭にコメント形式の定義を書くと、AviUtl2の設定UIと対応する変数を作成できる。

| 定義 | 用途 |
| --- | --- |
| `--track@` | トラックバー |
| `--trackgroup@` | 先に定義した2～3個のトラックバーのグループ |
| `--check@`、`--checksection@` | 通常またはセクション単位のチェック項目 |
| `--color@` | 色 |
| `--file@`、`--folder@` | ファイルまたはフォルダー |
| `--font@`、`--figure@` | フォントまたは図形 |
| `--select@` | リスト選択 |
| `--text@`、`--string@` | 複数行または1行の文字列 |
| `--value@` | 数値、文字列、配列 |
| `--data@` | 最大1024バイトの汎用データ領域 |
| `--group`、`--separator` | 設定UIのグループと区切り |
| `--label` | オブジェクト追加メニューの階層ラベル |
| `--filter` | `*.anm2`のフィルタオブジェクト対応 |
| `--script` | `luaJIT`または`lua`の指定。新形式の既定は`luaJIT` |
| `--information` | スクリプトの説明 |
| `--require` | 必要なAviUtl2本体バージョン |
| `--[[pixelshader@...]]` | HLSLピクセルシェーダー |
| `--[[computeshader@...]]` | HLSLコンピュートシェーダー |

項目名を`内部名::表示名`の形にすると、設定画面では最後の`::`より後だけを表示できる。トラックバー値は内部で32bit整数として保持され、指定範囲に応じて精度が調整される。

`--filter`を指定したスクリプトには、終了時にオブジェクトのサイズや変数を変えたままにしない、`obj.draw()`以降のフィルターが継続する、引数なしの`obj.effect()`が処理されない、などの制約がある。一般の`スクリプト制御`と同じ扱いにしない。

## 元の円運動の読解

### 対象

2026-07-24に最新状態を再取得し、変更前の選択中オブジェクトを確認した。

- シーン: Root
- レイヤー: 0
- フレーム: 9213～9413
- オブジェクト種別: `figure`
- 効果順: `図形`、`標準描画`、`スクリプト制御`
- 図形: 白い円、サイズ100
- 標準描画: X=`0.00`、Y=`-160.00`

`index`と`state_token`は一時値なので本書には保存しない。

### Lua本文

```lua
-- 円移動
l = 200

-- 半径
r = 360 * obj.time / 2

obj.ox = obj.ox + math.sin(r * math.pi / 180) * l
obj.oy = obj.oy - math.cos(r * math.pi / 180) * l
obj.rz = obj.rz + r
```

実際には`l`が半径、`r`が角度であるため、コメント名は入れ替わっている。処理内容は次のとおり。

- `l = 200`: 軌道半径200px
- `r = 360 * obj.time / 2`: 2秒で360度進む角度
- Xへ`sin`、Yへ負の`cos`を加算: 上から時計回りの円軌道
- `obj.rz`へ同じ角度を加算: 軌道と同期して1周期に1回転

今後AIが生成するときは、`local radius`、`local period`、`local angle`のように意味に合うローカル変数名を使用する。

| 9228 | 0.5 | `(200, 0)` | `(1159.47, 379.51)` |
| 9243 | 1.0 | `(0, 200)` | `(959.49, 579.47)` |

## 初回のLua本文変更: 横長の8の字運動

### 現在のLua本文

レイヤー0・9213～9413の選択中オブジェクトについて、`preview_set_object_parameters`で変更予定を確認してから`set_object_parameters`で次のLuaへ変更した。

```lua
-- 横長の8の字運動
local width = 240
local height = 120
local period = 4
local phase = obj.time * math.pi * 2 / period

obj.ox = obj.ox + math.sin(phase) * width
obj.oy = obj.oy + math.sin(phase * 2) * height
obj.rz = obj.rz + math.sin(phase) * 20
```

- Xは4秒周期、振幅240pxの正弦波である。
- YはXの2倍の周波数、振幅120pxの正弦波である。
- Xが1往復する間にYが2往復するため、中央で交差する横長の8の字軌道になる。
- Z軸回転はX位相と同期し、`-20`～`20`度の範囲で往復する。
- 変数はすべて`local`とし、他のスクリプト実行へ不要な状態を残さない。

### 本文の改行形式

最初に実改行を含む文字列を設定値として渡したところ、プレビューは成功したが、適用後の再読取値が要求値と一致せず`write_verification_failed`になった。Aul2MIRAIは元の円運動Luaを自動復旧し、再読取でも復旧を確認した。

各行をリテラルの`\n`で連結した値へ直して再プレビューすると`will_change=true`となり、適用後は`applied=true`、`changed_count=1`、`verified`と要求値の完全一致を確認した。

したがって、`スクリプト`設定をPipeで変更するときも、エイリアスと同様に改行をリテラルの`\n`で渡す。
